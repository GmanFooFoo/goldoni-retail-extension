# Pläne

Hier liegen die **konkreten Pläne**, wie wir das Projekt weiterbringen. Drei
Arten:

## 1. Rollout-Plan

Der [**Rollout-Plan**](rollout-plan.md) ist der **Zeitstrahl bis zum Launch**.
Er zeigt die Phasen, Abhängigkeiten und offenen Entscheidungen.

**Scope-Entscheidung (Stand 10.04.2026):** Phase 1 beschränkt sich auf
**Vakuum-Produkte**. Tiefkühl kommt in Phase 2 (nach erfolgreichem Vakuum-Launch).
Details dazu im [Rollout-Plan](rollout-plan.md).

## 2. Doc-Revisionspläne

Pro kritischem Dokument ein Plan, wie es von v1 auf v2 kommt. Datei-Schema:
`[doc-nummer]-v2-plan.md`, z.B. `02-v2-plan.md`.

Ein Revisionsplan enthält:

- **Was muss sich ändern?** (Liste der Findings)
- **Welche Infos brauchen wir noch?** (Silvio, Recherche, Labor, Behörde)
- **Wer liefert was, bis wann?**
- **Was bleibt gleich?** (damit klar ist, was nicht doppelt überarbeitet wird)

## 3. Backlog

Die fortlaufende technische Aufgabenliste liegt im Repo-Backlog (nur über
GitHub sichtbar, nicht in dieser Lese-Ansicht). Für dich (Silvio) reicht
der Rollout-Plan plus die Findings.
