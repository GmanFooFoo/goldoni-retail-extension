# Findings

Hier landen die **konsolidierten Ergebnisse** der Reviews. Kein "Marcus sagt
X, Pietro sagt Y" — sondern pro Dokument **ein** zusammengefasster Bericht,
sortiert nach Schwere:

1. **Stopp-Findings** (rot) — das geht so nicht live. Muss vor dem Launch korrigiert werden.
2. **Qualitäts-Findings** (gelb) — das geht live, ist aber schwach. Sollte verbessert werden.
3. **Nice-to-have** (blau) — Hinweise und Verbesserungsvorschläge.

Jeder Finding-Eintrag hat:

- **Was ist das Problem?**
- **Warum ist es ein Problem?** (Risiko, rechtlich, finanziell)
- **Wer hat es entdeckt?** (welche Persona(s))
- **Wie lösen wir es?** (Verweis auf Plan)

## Dateinamen-Schema

Pro Dokument eine Datei: `[doc-nummer]-findings.md`, z.B. `02-findings.md`
für die konsolidierten Findings zu Doc 02.

## Zusätzlich: Inconsistencies

Widersprüche **zwischen** Dokumenten (z.B. Doc 10 sagt "8 Wochen", Doc 13 sagt
"6 Wochen") werden in einer separaten Datei
[`inconsistencies.md`](inconsistencies.md) gesammelt. Die wird laufend
aktualisiert, während wir durch die Reviews gehen.
